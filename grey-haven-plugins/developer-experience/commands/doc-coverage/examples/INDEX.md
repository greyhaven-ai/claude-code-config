# Doc Coverage Examples

Complete examples demonstrating documentation coverage analysis workflows.

## Available Examples

### [coverage-metrics-examples.md](coverage-metrics-examples.md)
Comprehensive examples of documented vs. undocumented code across all 5 coverage categories.
- **Function Coverage** - JSDoc examples (100%, 50%, 0% documented)
- **Class Coverage** - Python docstring examples with attributes and examples
- **API Endpoint Coverage** - Complete endpoint documentation with all required fields
- **Type Coverage** - Interface and field-level documentation examples
- **Example Coverage** - Working code examples demonstrating usage

### [typescript-coverage-example.md](typescript-coverage-example.md)
Complete TypeScript project documentation coverage analysis.
- Full project scan with coverage percentages
- Module-by-module breakdown
- Critical gaps identification
- Auto-fix suggestions for TypeScript
- Integration with ESLint and TSDoc

### [python-coverage-example.md](python-coverage-example.md)
Complete Python project documentation coverage analysis.
- FastAPI and SQLModel project structure
- Docstring coverage analysis
- Pydantic model documentation patterns
- Auto-fix suggestions for Python
- Integration with Sphinx and pydocstyle

### [api-coverage-example.md](api-coverage-example.md)
REST API documentation coverage analysis.
- OpenAPI 3.1 compliance checking
- Endpoint documentation requirements
- Request/response schema validation
- Error response documentation
- Authentication and authorization docs
- Code example requirements

### [auto-fix-examples.md](auto-fix-examples.md)
Before/after examples of auto-generated documentation.
- **Function auto-fix** - JSDoc generation with inferred types
- **Type auto-fix** - Interface and field descriptions
- **Class auto-fix** - Class docstrings with attributes
- **API auto-fix** - OpenAPI documentation generation
- **Validation** - Testing generated documentation
